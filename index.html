Image Cash Letter (ICL) Tool

  • A tool designed to handle Image Cash Letter (ICL) files as part of the Cheque Truncation System (CTS) used by Philippine Clearing House Corporation (PCHC).
  • The ICL tool allows for reading of Inward, Outward, Returns, and Archive-only ICL files to view:
    • Front and rear check images in monochrome
    • Front check image in grayscale
    • Front ultra-violet (UV) check image for High-Value checks
    • ICL Record Data values
  • This can also be used to tag and generate Return Check(s) as Return ICL files to be submitted to Participants Bank Module (PBM)
  • Up-to-date Return Reason Codes and customizable Image Quality Assurance parameters
  • Absolutely free for qualified PCHC member banks!

Image Cash Letter (ICL) to eDesk File Converter

  • A cost-saving solution that converts Image Cash Letter (ICL) files to eDesk-compatible files (FIM/RIM/IDX) to help banks with implementing Philippine Clearing House Corporation's (PCHC) Check Image Clearing System (CICS).
  • This solution allows banks to continue to use their existing check verification systems that utilizes eDesk (FIM/RIM/IDX) or exported image files.
  • Time-saving advantage of directly converting ICL files to the exact format used by the exported image files generated by PCHC's Image Viewer
  • Better manpower management and process scheduling by providing Real-Time conversion. Convert the ICL files as they are received at the Participants Bank Module (PBM) from PCHC. This allows the banks to perform their check verification and other processes throughout the day without waiting to receive the converted files from PCHC at end of day.
  • Provides flexibility to banks by giving them time to develop/acquire solutions that will utilize the ICL files from CICS.

Straight-Through Processing for Electronic Clearing Systems

  • A result of more than 2 years of continuous research and interactions with the banking community, the Straight-Through Processing for Electronic Clearing Systems (STPC)
    is designed to handle the automated processing of transactions by the following Philippine Clearing House Corporation (PCHC) systems:
    • Electronic Peso Clearing and Settlement (EPCS)
    • Philippine Domestic Dollar Transfer System (PDDTS)

Features

  • No need for EPCS and PDDTS Front-end software.
  • Runs as a Windows service on either 32-bit and 64-bit Windows Operating Systems.
  • User-defined email alerts based on events and process statuses.
  • Failover switching in case one of PCHC's internet gateways is unavailable.
  • Built-in web portal for Administration and Monitoring.
  • Audit trails on system activities and user access.
  • Highly customizable to tailor fit Bank's Inward and outward file conversions and automations.
  • Transactions are processed and sent to PCHC as soon as they are available.
  • Increased productivity by eliminating manual sending of transactions to PCHC, allowing bank personnel to perform other tasks.

Check Image Clearing System (CICS) Front-End System

  • Client-Server model from Check Image Capturing, Data Entry, Balancing, ICL Generation, File Encryption to Transmission to Server.
  • We are the first to develop the Front-End Management System for the Philippine Clearing House back in June 2018 which is still being used today and is made available to partner banks at their head office.
  • Our FES solution features an all-in-one client for users to capture, enter data, balance amounts which then generates the Image Cash Letter (ICL) files that undergoes file and transmission encryption.
  • The client allows scanning of checks as well as deposit slips for balancing.
  • Unrecognized items are immediately tagged after scanning and prompts the user for rescanning.
  • ICL files are generated after technical verification and balancing and are transferred to the FES server for processing to PCHCs PBM.
  • ICL files undergo several layers of encryption for generating, transmitting, and processing.
  • Tested with no failure rates at network speeds as low as 64 Kbps to 512 Kbps.
  • We also offer a variety of FES client types:
    • FES Client for Branch/Store Regular Outward
    • FES Client for Branch/Store On-Us Outward
    • FES Client for On-Us/SOA processing
    • FES Client for Rural Banks with Inward processing
    • FES Client for Remote Check Deposits
